VMWare Workstation 启动异常

4/13/2022 vmware

# VMware Workstation 启动异常

# 01

# 报错内容

VMware (opens new window) Workstation 无法连接到虚拟机。请确保您有权限运行该程序、访问改程序使用的所有目录以及访问所有临时文件目录。未能将管道连接到虚拟机: 所有的管道范例都在使用中。

# 原因分析

在虚拟机运行过程中通过任务管理器强行关闭虚拟机。

# 解决思路

  1. + , 输入services.msc, 回车确认；

  2. 在“服务”中将有关VMware（4个）的启动类型全部改为禁用，重启计算机

  3. 再次回到之前服务配置的地方，再将启动类型全部改为自动；

  4. 启动虚拟机，已经没有刚才的问题，当然可能会出现新的的问题

  5. (option)之后出现了 “由于找不到VCRUNTIME140_1.dll,无法继续执行代码，重新安装程序可能会解决此问题”

# 02

# 报错内容

由于找不到 VCRUNTIME140_1.dll , 无法继续执行代码，重新安装程序可能会解决此问题。

# 原因分析

一般来说这配置文件缺失或者说这些文件错误，主要还是在安装电脑系统的时候安装不完全，或者说在安装时有错误而导致后面在安装某些软件需要用到这个一类的配置文件会出错从而导致安装失败。

# 解决思路

  1. 找到缺少的dll文件，并移动到相应的系统路径下

  • 相关地址

    • 主地址 (opens new window)
    • 下载地址 (opens new window)

  • 存放路径

    • x86 C:\Windows\SysWOW64
    • x64 C:\Windows\System32
  1. 重新启动虚拟机

# 03

# 报错内容

无法将网络更改为桥接状态：没有未桥接的主机网络适配器

# 解决思路

  1. 在VMare网络编辑器中,删除所有网络，并重启；
  2. + , 输入cmd, 回车确认；
  3. 在cmd命令行界面中输入“ncpa.cpl”, 回车确认；
  4. 选择WLAN或以太网，选择属性，安装->服务->找到Vmware，并选择“VMware Bridge Protocol”，确认；
  5. 在VMare网络编辑器中，选择恢复网络默认配置。
